## 一、项目概述
目标：开发前后端分离的简易题库生成管理系统，支持 AI 出题、手工出题、题库管理（增删改查）等功能。
然后还有学习心得展示、题库管理（分页/筛选/搜索）、手工/AI 出题、批量删除、AI 题目预览勾选等常见功能

## 二、技术
### 前端
- 核心框架：React 18
- 路由：React Router v6
- UI 组件库：Ant Design（antd）
- 脚手架：Vite
- 其他依赖：axios（接口请求）、react-markdown（Markdown 渲染）
- 启动命令：npm run dev（端口 5173）、npm run build（编译到 dist）

### 后端
- 开发语言：Go 1.20+
- Web 框架：Gin
- 数据库：SQLite（文件型数据库，无需额外安装）
- 第三方依赖：gorm（ORM 框架）、resty（HTTP 客户端，调用大模型 API）、godotenv（读取环境变量）
- 启动命令：go run .（端口 8080）

## 三、流程
### （1）AI 出题流程
1. 前端输入参数，如题型/数量/难度/编程语言→ 调用后端 `/api/ai-generate` 接口；
2. 后端构造 Prompt → 调用阿里云百炼的通义大模型 API → 解析返回结果（按约定格式提取题目内容/选项/答案）；
3. 前端接收生成的题目 → 打开预览弹窗 → 勾选需要的题目 → 点击确认 → 批量调用 `/api/questions` 接口存入数据库。

### （2）题库管理核心逻辑
- 分页：前端传递 page/size 参数，后端通过 `OFFSET (page-1)*size LIMIT size` 实现；
- 筛选/搜索：后端通过 SQL 条件查询（`type = ?` 筛选题型，`content LIKE %keyword%` 搜索内容）；
- 批量删除：前端收集勾选的题目 ID 数组 → 调用 `/api/questions`（DELETE 方法）→ 后端通过 `db.Delete(&Question{}, req.IDs)` 批量删除。

### （3）学习心得渲染流程
1. 后端 `/api/study-note` 接口读取根目录 `学习心得.md` 文件内容；
2. 前端调用该接口获取 Markdown 字符串 → 通过 `react-markdown` 组件渲染为 HTML 展示。
