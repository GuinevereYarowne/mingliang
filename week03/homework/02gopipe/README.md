# 日志处理系统（生产者-消费者模型）

## 功能说明
简单的日志处理系统，包含日志生成、过滤、存储三个核心阶段，支持特殊日志（ID=50）触发的退出机制。
1. 日志生成：1个协程循环生成100条日志（ID从1到100），并发送到日志通道。
2. 日志过滤：3个协程并发消费日志，筛选出偶数ID重要日志，发送到过滤通道；检测到ID=50的日志时，发送停止信号。
3. 日志存储：1个协程消费过滤通道的重要日志，打印日志详情模拟存储。
4. 退出：所有协程监听停止信号，收到信号后立即终止执行，避免资源泄漏。

## 代码结构

LogEntry  存储日志ID和内容
produceLogs 生成日志并发送到日志通道
filterLogs  并发过滤日志，筛选重要日志并发送信号
storeLogs   接收重要日志，模拟存储（打印）
main    调用函数，协调退出

## 相关附加小说明
logChannel和filteredChannel均设置缓冲大小为 10,减少 Goroutine 切换开销，提升并发效率。

仅由 "数据发送方" 或 "协调者" 关闭通道，避免多个 Goroutine 同时关闭通道导致的panic。

全局 wg：负责等待 "生产者" 和 "存储者" 两个核心 Goroutine，确保系统不会提前退出。
